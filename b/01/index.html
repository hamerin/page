개요

공개 키 암호란?

대칭 키 암호

대칭 키 암호 (Symmetric-key Cryptosystem) 는, 암호화 방식의 하나로, 암호화와 복호화에 같은 암호 키를 쓰는 방식을 의미한다. 따라서, 대칭 키 암호에서는 암호화를 하는 측과 복호화를 하는 측이 같은 암호 키를 공유해야 한다.

이 방식은 후술할 공개 키 암호 방식에 비해 크게 단순하며 빠르게 암호화 및 복호화가 가능하다는 장점이 있다. 그러나, 시간이 지남에 따라 암호화하여 주고받을 데이터가 많아지고, 그 대상들이 다양화되면서 각 사용자들이 모든 통신에 공통 암호 키를 주고받기 어려워지는 문제가 발생하였다.

공개 키 암호

공개 키 암호 (Public-key Cryptosystem) 는, 역시 암호화 방식의 하나로, 암호화와 복호화에 다른 암호 키를 사용한다. 같은 암호 키를 쓰는 대칭 키 암호에 비교하여, 비대칭 키 암호로도 불린다.

공개 키 암호에서는, 각 사용자들이 공개 키와 비밀 키를 각각 하나씩 가진다. 이 2개의 키는 상보성을 가지고 있어, 공개 키로 암호화된 평문은 비밀 키로만 복호화할 수 있고, 그 반대 역시 성립한다. 이에, 수신자가 발신자의 공개 키로 평문을 암호화하여 보내면, 발신자는 자신의 비밀 키로 암호문을 복호화시킬 수 있고, 이렇게 암호화된 암호문은 비밀 키를 가지고 있는 발신자를 제외한 어떤 사용자도 복호화시킬 수 없다.

이 방식을 통해, 암호 키의 상호 교환이 필수적이지 않게 되어, 기존 대칭 키 암호의 문제점을 해결할 수 있게 되었다. 하지만, 이 방식은 대칭 키 암호에 비해 복잡한 수학 계산이 요구되기 때문에, 암호화 밒 복호화가 상당히 느리다. 그러나, 이 문제점은 공개 키 암호를 통해 대칭 키 암호의 공통 암호 키를 교환함으로써, 충분히 해결될 수 있다.

RSA 암호

RSA 암호 는 공개 키 암호의 하나로, 최초의 전자서명이 가능한 공개 키 암호로 알려져 있다. 1978년, MIT의 Ron Rivest, Adi Shamir와 Leonard Adleman의 연구에 의해 체계화되었으며, RSA란 이름은 이들의 이름의 앞글자를 따 만들어졌다. 이 글의 대부분은 그들의 연구논문을 바탕으로 작성되었다.

RSA 암호의 공개 키와 비밀 키는 다음과 같은 방식으로 생성되고, 그 안정성은 임의의 수를 소인수분해하는 것이 NP 문제라는 것에 기반해 있다.

  2. 두 수를 곱하여 를 계산한다.

  4. 을 찾는다. (: 최대공약수)

암호화 및 복호화

키 에 대해, 평문/암호문을 양의 정수 의 나열로 변환 가능하다. 이때, 각 에 대해 다음과 같은 시행을 하여 암호화/복호화된 결과 을 얻는다.

이 과정을 통해, 평문을 암호화 후 복호화한 결과는 가 되고, 후술할 오일러 파이 정리에 의해 이 되어, 암호가 타당함을 알 수 있다.

이 암호에서 를 알고 있더라도 를 모르면, 을 구하는 것이 매우 어려우므로, 를 구해내는 것이 사실상 불가능하다. 따라서, RSA 암호는 공개 키 암호의 조건을 만족함을 알 수 있다.

Solovay-Strassen primality test

소수 와 모든 양의 정수 에 대해 다음이 무조건 성립한다. (르장드르 기호의 성질)

만약 가 합성수일 경우, 위의 식은 적어도 의 확률로 거짓이 된다.

이 알고리즘은 테스트 한 번 당 평균적으로 의 시간 복잡도를 가진다.

따라서, 임의의 에 대해 이 테스트를 번 반복하면 적어도 의 정확도를 보장받는다. 만 되더라도 이 테스트가 틀릴 확률은 이 되어 일반적으로 무시 가능해진다.

  _mul = 1
  for _ in range(e):
    _mul *= n
    _mul %= MOD

With Addition-chain exponentiation

def pow_fast(n, e, MOD):  # O(log e)
  _mul = n
  _res = 1
  for digit in str(bin(e))[2:][::-1]:
    if digit == '1':
      _res *= _mul
      _res %= MOD
    _mul *= _mul
    _mul %= MOD
  return _res

실행 시간:

xxxxxxxxxx
1.79 µs ± 5.59 ns per loop (mean ± std. dev. of 7 runs, 1000000 loops each)

# 잉여역수 구하기 (Extended Euclidian Algorithm)
def get_arithmetic_inv(r1, r2, s1=1, s2=0, t1=0, t2=1):
  if r1 < r2:
    r1, r2 = r2, r1
  r = r1 % r2
  q = r1 // r2
  s = s1 - s2 * q
  t = t1 - t2 * q
  return t2 if r == 0 else get_arithmetic_inv(r2, r, s2, s, t2, t)

Solovay-Strassen primality test

def primality_test(p, loops):  # Solovay-Strassen Primality Test
  for _ in range(loops):
    r = random.randint(1, p - 1)
    if not (gcd(r, p) == 1 and jacobi(r, p) % p == pow_fast(r, (p - 1) // 2, p)):
      return False
  return True

def encrypt_message(key_public, message):
  ascii_message = base64.b64encode(message.encode()).decode()
  message_number = list(map(ord, ascii_message))
  encrypted = list(
      map(lambda x: str(apply_key(x, key_public)).rjust(len(str(key_public[1])), '0'), message_number))
  return ' '.join(encrypted)
